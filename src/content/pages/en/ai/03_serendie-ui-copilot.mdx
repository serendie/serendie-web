<PageHeadLine number={0}>What is Serendie UI Copilot</PageHeadLine>

To achieve high-quality AI-powered application development (Agentic Coding), **well-structured designs** are essential. This requires correctly using design tokens and UI components, which in turn demands familiarity with the Serendie Design System (SDS).

Serendie UI Copilot is a **Figma plugin that supports UI design using the Serendie UI Kit**. Its AI-powered consultation and validation/fix features help you create UI designs that follow SDS guidelines.

It is designed for two main scenarios:

- When you need precise designs intended for handoff to engineers (for UI designers)
- When adding new components to the Serendie UI Kit (for SDS maintainers)

Internally, the plugin uses the [Serendie MCP Server](/en/ai/mcp-server) and functions as an AI agent specialized for these scenarios.

</PageSection>

<PageHeadLine number={1}>Features</PageHeadLine>

### Ask AI

You can ask about UI component design using the Serendie UI Kit through a chat interface.

- "Which design token should I use for the text on this screen?"
- "How should I use the different properties of the Button component?"
- "Can you review the component I'm working on?"
- "I'd like help with naming this component."
- "What properties does a Slider component need? Please research other design systems."

These are typical questions. The AI answers while referencing Serendie Design System guidelines and assets. Since Figma context such as the currently selected frame is shared with the AI, you can ask questions immediately without explaining the situation in detail.

For component design, it's also important to follow UI Kit standards beyond just Serendie UI. Serendie UI Copilot provides advice on specific improvements and best practices while referencing related design systems such as [Material Design 3](https://m3.material.io/) and [Ark UI](https://ark-ui.com/).

### Validation & Fix

Serendie UI is built on design token rules such as [color roles](/en/foundations/color/role/) and [typography roles](/en/foundations/typography/#section-4). Following these rules ensures accessibility, including proper contrast ratios. The validation feature checks whether selected elements in Figma follow the design token rules.

### Ask AI

1. Select the target element in Figma (this is shared with the AI as context)
2. Choose from suggested questions or enter your own question

### Validating & Fixing Designs

1. Select the element you want to validate in Figma
2. Click the "Validate" button to display design token validation results
3. Detect areas where components can be used (AI analysis may take time depending on screen complexity)
4. Click "Apply Components" or "Fix Design Tokens" to auto-fix

_＜＜＜Insert image (screenshot)＞＞＞_

Validation results are displayed with three types of icons:

- Error: Parts that don't follow design token rules and require correction
- Warning: Parts that need confirmation, such as missing design token settings
- Normal: No issues

Click on a validation result item to select the corresponding element in Figma, making it easy to see where the issue is.

### The "Ask AI" button doesn't appear

The OpenAI API key may not be configured. Enter the API key from the settings icon in the top right of the plugin.

### About data handling

When you use the Ask AI feature, layer information of the selected element, text information from validation results, and chat conversation content are sent to the OpenAI API. Under OpenAI's API policy, **data sent via the API is not used for AI model training.** For the latest information, please check [OpenAI's Terms of Use](https://openai.com/policies/terms-of-use).
