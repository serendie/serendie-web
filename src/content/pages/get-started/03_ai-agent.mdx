SerendieデザインシステムとMCPサーバー、デザインからコードへの変換ツールについて。

C:MCPサーバーを使うことで、AIエージェントに対してSerendieデザインシステムを理解させることができます。これにより、デザイナーとデベロッパーが同じ言語で会話できるようになり、デザインとコードの乖離を防ぐことができます。

</PageSection>

<PageSection>
  <PageHeadLine number={0}>MCPとは</PageHeadLine>

Model Context Protocolは、AIエージェントが外部データソースやサービスに安全に接続するためのオープンプロトコルです。MCPを使うことで、AIエージェントはSerendieデザインシステムの情報にアクセスし、デザイントークンやコンポーネント情報を取得できます。これにより、デザインとコードの一貫性を保ちながら、AIエージェントによる開発支援が可能になります。

<PageARef href="https://modelcontextprotocol.io/introduction">
  What is the Model Context Protocol? - Model Context Protocol
</PageARef>

</PageSection>

SerendieデザインシステムのMCPサーバーはウェブサイトのサブパスとして提供しており、コンテンツは随時更新されます。シンプルな仕組みのため、MCPサーバーを立ち上げる必要はありません。

## デザインから実装する

Figma Dev Mode MCPサーバーとSerendieデザインシステムのMCPサーバーを使うことで、FigmaにてSerendieデザインシステムのコンポーネントを使って作成した画面を、そのままコードに変換することができます。

Figma Dev Mode MCPサーバーは、Figmaのデザインファイルを読み取り、そこに含まれるコンポーネント情報を取得します。

デザインシステムとしては、デザイン要素が1対1でコードに変換できることが理想ですが、現実的には難しいことが多いです。
